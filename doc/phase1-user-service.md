# User Service — Implementation Details

## Tech Choices

| Concern | Library | Why |
|---|---|---|
| Web | Spring Boot 3 + Spring Web | Standard REST |
| Security | Spring Security 6 | Filter chain, password encoding |
| ORM | Spring Data JPA + Hibernate | Repository pattern |
| DB migrations | Flyway | Schema versioning per service |
| JWT | JJWT 0.12 (jjwt-api/impl/jackson) | Mature, type-safe JWT builder |
| Validation | Jakarta Bean Validation | `@Valid` on request DTOs |
| Boilerplate | Lombok | Reduces getter/setter noise |

## Database Schema (Flyway V1)

```sql
users           -- driver accounts
vehicles        -- plates linked to a user
refresh_tokens  -- stored refresh tokens (UUID, expires_at)
```

All PKs are `UUID` generated by PostgreSQL (`uuid_generate_v4()`).
Foreign keys use `ON DELETE CASCADE` so deleting a user removes their tokens and vehicles.

## Security Filter Flow

```
Request
  │
  ▼
JwtAuthFilter (OncePerRequestFilter)
  ├── /auth/** → skip, pass through
  └── other   → extract Bearer token
                  │
                  ├── valid   → set SecurityContext principal = UUID(sub)
                  └── invalid → SecurityContext empty → 403 from Spring Security
```

`@AuthenticationPrincipal UUID userId` in controllers reads the principal set by this filter.

## Error Responses

All errors use Spring 6's `ProblemDetail` (RFC 9457):

```json
{
  "type": "about:blank",
  "title": "Conflict",
  "status": 409,
  "detail": "Email already registered"
}
```

| Scenario | HTTP Status |
|---|---|
| Validation failure | 400 |
| Duplicate email / plate | 409 |
| Wrong credentials / bad token | 401 |
| User not found | 404 |

## API Endpoints

```
POST /auth/register        body: {name, email, password, phone?}
POST /auth/login           body: {email, password}
POST /auth/refresh         body: {refreshToken}

GET  /users/me             header: Authorization: Bearer <token>
POST /users/me/vehicles    header: Authorization, body: {plate, make?, model?}
GET  /users/me/vehicles    header: Authorization
```
